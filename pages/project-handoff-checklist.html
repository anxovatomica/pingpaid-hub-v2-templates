📦 Project Handoff Checklist

Complete handoff · 20 items · Professional delivery · Updated June 2026

A professional handoff is the difference between a one-time client and a repeat client. Here's everything to deliver.

⚠️ Important: Never hand off without final payment. Include a note: "All deliverables will be transferred upon receipt of final payment."

📁 Deliverables

All source files
Raw files, not just exports (PSD, Figma, AI, code repos, etc.)
Final exported files
High-res, web-optimized, or production-ready formats
Documentation
How to use, maintain, or update the deliverables
Style guide or brand guidelines
Colors, fonts, logo usage, voice/tone if applicable

🔐 Access & Credentials

All login credentials transferred
Hosting, domains, CMS, tools, accounts created during project
Ownership transferred
Domains, hosting, software licenses transferred to client
Admin access revoked
Your personal accounts removed, client given full admin access
Backup created
Client has a backup of everything before you remove your access

📋 Documentation

User manual or guide
How to use what you built (especially for CMS, custom tools, etc.)
Technical documentation
API docs, code comments, architecture diagrams if applicable
Troubleshooting guide
Common issues and how to fix them
Contact information for support
Your support terms, response time, and how to reach you

🤝 Professional Close

Final invoice sent
Clear breakdown of all charges, payment due date
Thank you email
Express gratitude, summarize what was delivered, offer next steps
Testimonial request
Ask for feedback and a testimonial while they're happy
Referral request
"Do you know anyone else who might need [service]?"
Future work discussion
Maintenance, retainer, or Phase 2 conversation started
Support period defined
30-day support included? What's covered vs. billable?
Handoff call scheduled
Walk them through everything live, answer questions in real-time
Final payment received
Never transfer final deliverables before payment is confirmed
